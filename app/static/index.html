How this works

Trains follow a fixed schedule, moving between stations along track segments at known travel times. The admin can report a problem — such as a closed track, a speed restriction, or a delayed train — using the controls below. The system instantly recomputes a new safe plan so that no two trains can ever be on the same track at the same time. The dashed “ghosts” on the map show the predicted new plan before it is applied, so you can review the impact first. Every decision the system makes is explained step by step in the Decision log below.

Report a problem

Each row is one action: choose the track or train it applies to, then click. The system previews a safe new plan before you apply it.

Close a track
Shuts this track completely — trains reroute around it or wait.
Block a track
Temporary obstruction — same routing effect as a closure until cleared.
Reduce speed on a track
Trains cross this track slower, so they arrive later.
Delay a train
This train departs later than planned.
Cancel a train
This train does not run at all.
Restrict a train from a track
Only this train avoids that track (it reroutes, or strands if it has no other route). Other trains keep using it.
Add a new train
Schedules a brand-new train collision-free against existing traffic (holding or rerouting as needed), or reports if it cannot be placed.
Clears all reported problems and added trains, returning to the original plan.
